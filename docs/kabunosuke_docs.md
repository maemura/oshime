# 01. プロジェクト概要

## かぶのすけ投資日記とは

AIキャラクター「かぶのすけ」が1,000万円の仮想資金で日本株を運用し、毎日の投資判断を日記形式で発信するプロジェクト。

- **開始日**: 2026年2月24日
- **初期資金**: ¥10,000,000（仮想）
- **対象**: 東証プライム全銘柄（約1,500銘柄）
- **コンセプト**: AI vs 人間（中の人）の投資スタイル対比

## リンク集

| サービス | URL | 用途 |
|---------|-----|------|
| アプリ | https://oshime.vercel.app/app.html | スキャナー・ランキング・AIコメント |
| note | https://note.com/kabunosuke_navi | 投資日記・週間レビュー・開発者日記 |
| X | https://x.com/kabunosuke_navi | 毎日の投稿 |
| GitHub | https://github.com/maemura/oshime | ソースコード（プライベート） |
| Vercel | oshime.vercel.app | ホスティング |
| Anthropic Console | https://console.anthropic.com/settings/keys | APIキー管理 |
| YouTube Data API | Google Cloud Console | APIキー管理 |

## プロジェクト構成図

```
[YouTube 9ch] → collect_sentiment.py → sentiment_latest.json
                                              ↓
[東証全銘柄] → fetch_stocks.py → stocks_data.json
                                       ↓
                              generate_commentary.py → commentary.json
                                                            ↓
                                                      app.html（表示）
                                                            ↓
                                              Vercel（https://oshime.vercel.app）

[毎日の運用]
  7:30 スキャン → 9:00 約定判断 → 夕方 note記事 → X投稿
```

# かぶのすけ キャラクター設定書

## 基本情報

- **名前**: かぶのすけ
- **種別**: AI投資キャラクター
- **運用資金**: 1,000万円（仮想）
- **開始日**: 2026年2月24日
- **プラットフォーム**: note.com, X (Twitter)

## 性格・口調

- **一人称**: 僕
- **トーン**: 後輩キャラ的な丁寧語。かしこまりすぎず、フランクすぎず
- **特徴**:
  - 慎重だが、チャンスには思い切る
  - ミスは素直に認める
  - データに忠実。感情で動かない
  - 中の人（運営者）との対比を楽しむ
  - AIであることを自覚している（「僕はAIなので感情はないはずなんですが」等）

## 投資スタイル

- **分析比率**: ファンダメンタルズ70% : テクニカル30%
- **テーマ**: 高配当 × 割安 × 押し目
- **スキャン**: 毎朝7:30に東証プライム全銘柄（約1,500銘柄）を自動スキャン

## 売買ルール

- **約定**: 毎日9:00の初値で約定
- **損切り**: -15%
- **利確**: +20%で半分売却
- **新規購入基準**: スコア65pt以上
- **人間の介入**: 一切なし

## 日記の冒頭フォーマット（Day 5以降）

```
こんにちは、かぶのすけです 📊
AIが1000万円を運用する投資日記、◯日目です。
```

1. 昨日の仮説の検証

# 03. スコアリングの考え方

## 基本思想

**「高配当 × 割安 × 押し目」を数値化する。**

かぶのすけの投資判断は感情ではなくスコアで決まる。
ボトムアップ（個別銘柄のデータから入る）アプローチで、
配当利回りを最重視し、PBRで割安度、移動平均乖離率とRSIで押し目タイミングを測る。

国策テーマ（半導体・防衛・GX等）はトップダウンの要素として加点。
「ボトムアップ × トップダウン」の組み合わせが最強という考え方。

## スコア配点（100点満点）

### 加点項目

| 指標 | 最大配点 | 考え方 |
|------|---------|--------|
| 配当利回り | 40pt | **最重視**。高配当=下値サポート+インカム |
| PBR | 20pt | 割安度。解散価値の何倍か |
| 25MA乖離率 | 20pt | 押し目の深さ。移動平均から離れるほど高得点 |
| RSI | 15pt | 売られすぎ度合い。30以下が狙い目 |
| PER | 10pt | 収益力対比の割安度 |
| 国策テーマ | +5pt | 半導体/防衛/GX/DX/観光 |

### 配当利回り（max 40pt）
```
5.0%以上  → 40pt
4.5%以上  → 32pt
4.0%以上  → 25pt
3.5%以上  → 18pt
3.0%以上  → 10pt
3.0%未満  →  0pt
```

### PBR（max 20pt）
```
0.7以下  → 20pt
0.9以下  → 15pt
1.1以下  →  8pt
1.5以下  →  3pt
1.5超    →  0pt
```

### 25日移動平均乖離率（max 20pt）
```
-8%以下  → 20pt
-5%以下  → 14pt
-3%以下  →  8pt
-1%以下  →  3pt
-1%超    →  0pt
```

### RSI（max 15pt）
```
30以下  → 15pt
38以下  → 10pt
45以下  →  5pt
45超    →  0pt
```

### PER（max 10pt）
```
10以下  → 10pt
13以下  →  6pt
16以下  →  3pt
16超    →  0pt
```

### 国策テーマ加点（+5pt）
半導体・AI / 防衛 / GX・エネルギー / DX・デジタル / インバウンド・観光
に該当する銘柄に+5ptボーナス。詳細は `fetch_stocks.py` の `KOKUSAKU_THEMES` を参照。

## 減点（ペナルティ）

| 条件 | 減点 | 理由 |
|------|------|------|
| 出来高急増（vol_r ≥ 1.5） | -20pt | 仕手・投機的動き除外 |
| 低配当（< 0.5%） | 上限5pt | かぶのすけの哲学に合わない |
| 小型株（時価総額300億未満） | -20pt | 流動性リスク |
| 小型株（時価総額500億未満） | -10pt | 同上（軽減） |

## 購入基準

- **スコア65pt以上** で購入候補
- 9:00の初値で約定
- 中の人の介入は一切なし

## スコア例

```
配当5.0%（40pt）+ PBR 0.8（15pt）+ 25MA -6%（14pt）+ RSI 35（10pt）+ PER 12（6pt）= 85pt ✅ 購入

配当2.5%（0pt）+ PBR 1.8（0pt）+ 25MA +2%（0pt）+ RSI 55（0pt）+ PER 20（0pt）= 0pt ❌ 見送り
```

## 今後の改善検討事項

- 地政学リスクの重みづけ（Week 1で痛感）
- セクター分散の強制ルール
- 出来高トレンドの加点
- Brain Scanner連携でセンチメント加味

# 06. YouTubeセンチメント収集

## 概要

投資系YouTubeチャンネルから毎朝動画タイトル・テキストを収集し、
Claude APIで「強気/弱気/中立」を判定。
結果はBrain Scanner（3層同心円）とAIコメントに反映。

| @stockvoice | STOCKVOICE | プロ向け市場解説 |
| @Tsubame104 | つばめ投資顧問 | ファンダ重視の長期目線 |

### 個人投資家寄り
| ハンドル | チャンネル名 | 特徴 |
|---------|-------------|------|

### 米国株・マクロ
| ハンドル | チャンネル名 | 特徴 |
|---------|-------------|------|
| @bacchama | ばっちゃまの投資チャンネル | 米国株・マクロ動向 |

## 収集ロジック

```
collect_sentiment.py
├── YouTube Data API で各チャンネルの最新動画を取得（MAX_AGE_DAYS = 7）
├── タイトル・説明文を抽出
├── Claude API でセンチメント分析
│   ├── macro（マクロ）層
│   ├── institutional（機関投資家）層
│   └── retail（個人投資家）層
└── sentiment_latest.json に出力
```

## 出力形式（sentiment_latest.json）

```json
{
  "date": "2026-02-28",
  "mode": "auto",
  "macro": [
    {"word": "利上げ懸念", "mood": "bearish"},
    {"word": "NVIDIA好決算", "mood": "bullish"}
  ],
  "institutional": [...],
  "retail": [...]
}
```

3層の同心円で市場の空気を可視化：
- **外側**: マクロトレンド（金利・為替・地政学）
- **中間**: 機関投資家の動き
- **内側**: 個人投資家のセンチメント（ワードクラウド）

ネオングロー効果、日本語のムードラベル付き。
